Learning AngularJS (ENG Only)

18 Apr 2014

When approaching for the first time to a new technology, I think sometimes it's essential to read a bit before you put your hands on the code. This especially when dealing with frameworks of a certain complexity rather than just simple libraries. Documentation, tutorials, blogs and any other sources of information on the web are good to start with.

This days I'm starting to work with AngularJS. After reading a lot of documentation and blog posts, and after doing some tutorials, I decided to write a summary of the best-practices I collected during the readings.

Note: this is neither a guide nor a tutorial that goes well to learn AngularJS. It's just a collection of best-practices from different sources, which seemed useful to put together. Base concepts are not explained: I collected only the tips that I found interesting. If you want to learn AngularJS, I suggest you to read the references

What is AngularJS

AngularJS is not a library. It is a JavaScript framework that embraces extending HTML into a more expressive and readable format. It decreases emphasis on directly handling DOM manipulation from the application logic, allowing for easier testing. It employs efficient two-way data binding and sensible MVC implementation, reducing the server load of applications. It features directives, which are incredibly robust tools that are significant contributors to Angular’s ubiquity. [1]

Concepts

A list of the main concepts of AngularJS: [2]

  • Template: HTML with additional markup
  • Directives: extend HTML with custom attributes and elements (like ng-app)
  • Model: the data that is shown to the user and with which the user interacts
  • Scope: context where the model is stored so that controllers, directives and expressions can access it
  • Data binding: Sync data between the model and the view
    • Angular provides a two-way data binding: whenever the input values change, the value of the expressions are automatically recalculated and the DOM is updated with their values.
  • Expression: access variables and functions from the scope (like `` )
  • Filter: formats the value of an expression for display to the user (like ``)
  • View: what the user sees (the DOM)
  • Controller: the business logic behind views
  • Dependency Injecton: Creates and wires objects/functions
  • Injector dependency injection container
  • Module configures the Injectior
  • Service reusable business logic independent of views

What to do if you want to...

... write view independent logic

When the application grows it is a good practice to move view independent logic from the controller into a so called service, so it can be reused by other parts of the application as well. [2]

... keep your code simple and modular

Split the controllers into different modules based on different pages and keep separated common filters and services. Make sure all modules have their dependencies set up correctly and components are succesfully injected. If you do not want to go through the refactor, learn and plan your modules before you start.[3]

... access the DOM

In Angular, the only place where an application touches the DOM is within directives. This is good as artifacts that access the DOM are hard to test. If you need to access the DOM directly you should write a custom directive for this. [2]

... test your code

Angular makes easy to test your code, but with a condition: you have to use its components (controllers, services, directives...) in the right way (e.g. if modify the DOM from a controller, it becomes tricky to test the views).

Use unit tests on controllers, integration tests to make sure the components works together, and E2E tests to test the user interaction.

Angular Expressions vs. JavaScript Expressions

  • Context: JavaScript expressions are evaluated against the global window. In Angular, expressions are evaluated against a scope object.
  • Forgiving: In JavaScript, trying to evaluate undefined properties generates ReferenceError or TypeError. In Angular, expression evaluation is forgiving to undefined and null.
  • No Control Flow Statements: you cannot use the following in an Angular expression: conditionals, loops, or exceptions.
  • Filters: You can use filters within expressions to format data before displaying it.

If you want to run more complex JavaScript code, you should make it a controller method and call the method from your view. [4]
